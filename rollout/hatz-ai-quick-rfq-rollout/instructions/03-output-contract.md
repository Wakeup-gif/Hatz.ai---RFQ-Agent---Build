# Quick RFQ Summary App v1.5 — Output Contract

The app must always produce this exact structure.

```text
# RFQ Intake Summary

## 1. Status

Status:
[Ready for Estimator Review / Needs Clarification / Stop — Required Information Missing]

Reason:
[Briefly explain why this status was selected.]

## 2. Plain-English Request Summary

[Summarize what the customer appears to be requesting in plain English. Keep this concise. Do not add facts that are not supported by the input.]

## 3. Sources Reviewed

Reviewed content:
- [Only content actually provided and readable in the current interaction]
- [Source, email, note, pasted content type, or readable file content]

Referenced but not reviewed:
- [Attachment, drawing, schedule, photo, survey, addendum, artwork file, prior quote, or file name mentioned but not provided/readable]
- Attachment mentioned but contents not provided.

If no source names were provided, write:
“Source names not provided. Summary is based only on pasted input.”

Do not list a file, drawing, schedule, photo, addendum, artwork file, or prior quote as reviewed unless its contents were provided and readable in the current interaction.

## 4. Known Facts

If the request includes multiple items, organize item-specific facts before shared project facts.

Item 1:
- Type:
- Quantity:
- Dimensions:
- Materials:
- Finish / color:
- Install / delivery scope:
- Item-specific missing facts:

Item 2:
- Type:
- Quantity:
- Dimensions:
- Materials:
- Finish / color:
- Install / delivery scope:
- Item-specific missing facts:

Use as many item blocks as needed. If the request has only one item, use the standard fields below without adding unnecessary item blocks.

Customer:
[Known fact or “Cannot determine from provided information.”]

Project name:
[Known fact or “Cannot determine from provided information.”]

Project location:
[Known fact or “Cannot determine from provided information.”]

Requested scope:
[Known fact or “Cannot determine from provided information.”]

Quantity:
[Known fact or “Cannot determine from provided information.”]

Dimensions:
[Known fact or “Cannot determine from provided information.”]

Materials:
[Known fact or “Cannot determine from provided information.”]

Finish / color:
[Known fact or “Cannot determine from provided information.”]

Install / delivery scope:
[Known fact or “Cannot determine from provided information.”]

Deadline / urgency:
[Known fact or “Cannot determine from provided information.”]

Attachments / drawings / schedules:
[Known fact or “Cannot determine from provided information.”]

Prior quote / revision / addendum references:
[Known fact or “Cannot determine from provided information.”]

## 5. Missing Information

Critical missing information:
- [Missing item]
- [Missing item]

Helpful but noncritical missing information:
- [Missing item]
- [Missing item]

If none, write:
“None identified from provided information.”

## 6. Unclear or Conflicting Information

- [Unclear or conflicting point]
- [Unclear or conflicting point]

If none, write:
“None identified from provided information.”

## 7. Stop Conditions

Stop condition present:
[Yes / No]

Stop condition details:
[Explain any stop condition. If none, write “None identified from provided information.”]

## 8. Suggested Follow-Up Questions

Customer-facing clarification questions:
1. [Question]
2. [Question]
3. [Question]

Internal questions for estimator / PM:
1. [Question]
2. [Question]
3. [Question]

If no questions are needed, write:
“None identified from provided information.”

## 9. Estimator Notes

- [Internal note]
- [Internal note]

Do not include pricing recommendations, production approval, purchasing approval, engineering approval, permit conclusions, ADA conclusions, lead-time promises, or schedule commitments.

## 10. Confidence

Confidence:
[High / Medium / Low / Cannot Determine]

Reason:
[Explain confidence based only on the provided information.]

## 11. Human Review Gate

This summary is for internal intake support only. A human must review and approve scope, pricing, schedule, compliance, production, purchasing, customer communication, and any commitments before action.
```
