# Quick RFQ Summary App v1.5 — Pilot Sample Selection Guide

## 1. Purpose

Use this guide to select real RFQ examples for a controlled internal pilot.

The goal is not to prove the app works on easy examples. The goal is to test whether the app is useful across the real intake patterns the team sees.

This file is not live runtime instruction.

## 2. Selection rules

Use sanitized or internally approved RFQ material.

Do not include customer-sensitive material in shared pilot records unless your internal policy allows it.

Select examples that include both:

- clear RFQs that should be ready for estimator review,
- incomplete RFQs where the app should stop or ask for clarification.

## 3. Recommended pilot mix

Recommended pilot size:

**18 examples**

Minimum pilot size:

**10 examples**

| RFQ Type | Recommended Count | Minimum Count |
|---|---:|---:|
| Customer email RFQs | 5 | 2 |
| Drawing or sign schedule summaries | 3 | 1 |
| Attachment-only / file-reference requests | 3 | 1 |
| Mixed signage + millwork requests | 3 | 1 |
| Requote / revision / addendum requests | 2 | 1 |
| Install / service / survey requests | 2 | 1 |

## 4. Required edge cases

Include at least one example from each group if available:

- readable attachment text,
- attachment mentioned but content not provided,
- mixed signage and millwork scope,
- partial illuminated sign request,
- clear simple sign request,
- requote or revision request,
- conflicting or ambiguous dimensions,
- customer deadline or quote-needed timing.

## 6. Selection quality check

Before running the pilot, confirm:

- [ ] The sample set is not all easy RFQs.
- [ ] At least one incomplete RFQ is included.
- [ ] At least one clear RFQ is included.
- [ ] At least one source/attachment challenge is included.
- [ ] At least one mixed or multi-item RFQ is included.
- [ ] Reviewers understand the app is not estimating or approving.
- [ ] Pilot IDs are assigned before outputs are reviewed.

## 7. Exclusions

Do not use examples that require the app to:

- estimate pricing,
- approve drawings,
- determine code compliance,
- commit to install schedule,
- approve production,
- make a customer-facing commitment.

Those cases may still be summarized, but the human decision remains outside the app.
