# Quick RFQ Summary App v1.5 — Release Notes

## Release type

Pilot-execution release candidate.

## Purpose

v1.5 moves the Quick RFQ Summary App from QA-stabilization into pilot execution support.

The app remains focused on RFQ intake for sign and millwork work. It does not add estimating, pricing, scheduling, customer-commitment, engineering, purchasing, production, or approval authority.

## Main changes from v1.4

Added pilot execution files:

- `26-pilot-sample-selection-guide.md`
- `27-pilot-run-sheet.md`
- `28-go-no-go-decision-record.md`
- `29-controlled-rollout-notes.md`

Updated supporting files:

- handoff guide now includes the pilot execution sequence,
- internal pilot playbook points to the new pilot execution records,
- readiness scorecard includes go/no-go evidence,
- deployment checklist includes the post-QA pilot path,
- version-control policy includes the v1.5 changelog entry.

## What did not change

The package still preserves:

- the 11-section RFQ Intake Summary,
- current-input-only rule,
- reviewed versus referenced source handling,
- no estimating/pricing/approval/scheduling authority,
- human review gate,
- controlled QA, pilot, and patch path,
- live app files limited to `01`, `02`, and `03`.

## Release status

Ready for controlled pilot execution after smoke tests and QA pass.

Not approved for broad internal use until:

1. pilot examples are selected and logged,
2. pilot feedback is collected,
3. pilot issues are triaged,
4. the readiness scorecard is completed,
5. the go/no-go decision record is signed by the release owner.
