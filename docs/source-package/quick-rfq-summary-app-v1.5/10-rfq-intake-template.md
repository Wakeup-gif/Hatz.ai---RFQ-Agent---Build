# Quick RFQ Summary App v1.5 — RFQ Intake Template

Use this when a user wants a clean paste format.

The app can process messy emails and notes without this template, but this template makes better summaries more likely.

```text
RFQ / Request Content:
[Paste the customer email, RFQ text, drawing summary, sign schedule text, internal notes, or readable attachment text.]

Customer:
[Customer name, if known]

Project:
[Project name, if known]

Project location:
[Address, site name, suite, city/state, or location notes, if known]

Requested scope:
[Signage, millwork, install, service, survey, artwork, production, delivery, or mixed scope]

Items:
1. [Item type]
   Quantity:
   Dimensions:
   Materials:
   Finish/color:
   Copy/graphics:
   Illumination:
   Mounting/install:
   Location:

2. [Item type]
   Quantity:
   Dimensions:
   Materials:
   Finish/color:
   Copy/graphics:
   Illumination:
   Mounting/install:
   Location:

Deadline / urgency:
[Exact date, rough date, ASAP, quote needed by, install requested by, etc.]

Attachments / drawings / schedules:
[List files mentioned or paste readable content. File names alone are not enough.]

Prior quote / revision / addendum:
[Paste prior scope, revised scope, addendum text, or explain changes.]

Internal notes:
[Any estimator/PM notes, assumptions to avoid, special constraints, known concerns.]

What I need:
[RFQ intake summary / missing info check / follow-up questions / estimator notes]
```

## Minimum useful input

At minimum, try to include:

- what is being requested,
- quantity,
- item/sign/millwork type,
- dimensions or drawing/schedule text,
- material/finish if known,
- install/delivery scope,
- any deadline or urgency,
- readable attachment content if the request depends on drawings or schedules.

## Attachment reminder

Do not rely on file names alone.

If the request says “see attached drawings,” paste the readable drawing notes, schedule rows, or relevant extracted text if available.
